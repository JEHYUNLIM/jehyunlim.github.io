"순수함수와 부수효과"

함수형 프로그래밍은 순수함수로 구성된 불변 프로그램 구축을 전제로 한다. 순수함수의 특성은 다음과 같다.

- 주어진 입력에만 의존할 뿐, 평가 도중 또는 호출 간 변경될 수 있는 숨겨진 값이나 외부 상태와 무관하게 작동한다.
- 전역 객체나 레퍼런스로 전달된 매개변수를 수정하는 등 함수 스코프 밖에서 어떠한 변경도 일으키지 않는다.

위 특성을 만족하지 않는 함수는 모두 '불순 impure'하다고 볼 수 있다. 명령형 프로그래밍에서는 변수 variable가 한 구문에서 다른 구문으로 옮겨지면서 그 값이 변하는 것이 기본이다.

var counter = 0;
function increment() {
  return ++counter;
}

이 함수는 자신의 스코프에 없는 외부 변수 counter를 읽고 수정하므로 불순하다. 일반적으로 외부 자원을 상대로 데이터를 읽고 쓰는 함수는 부수효과를 동반한다. counter는 암시적 implicit 전역 변수(브라우저 기반 자바스크립트에서는 Window 객체)를 통해 접근한다. this 키워드를 거쳐 인스턴스 데이터에 접근하는 것 역시 부수효과가 유발된다. 특히 자바스크립트의 this는 해당 함수의 런타임 context에 따라 달라지므로 여타 프로그래밍 언어와는 작동 방식이 다르다. 그래서 가급적 this는 쓰지 않는다. 부수효과가 발생하는 상황은 다음과 같이 다양하다.

- 전역 범위에서 변수, 속성, 자료구조 등을 변경
- 함수의 원래 인수 값을 변경
- 사용자의 입력을 처리
- 예외를 일으킨 해당 함수가 붙잡지 않고 catch 그대로 예외를 던짐 throw
- 화면 또는 로그 파일에 출력
- HTML 문서, 브라우저 쿠키, DB에 쿼리

역동적으로 움직이며 온갖 변경이 난무하는 프로그램에서 순수함수를 사용하기란 현실적으로 어려울 수 있지만, 실제로 함수형 프로그래밍은 모든 상태 변이를 근절하는건 아니고, 상태 변이를 줄이고 관리할 수 있는 프레임워크를 제공하여 순수함수와 불순함수를 구분한다. 불순한 코드는 밖에서 가시적인 부수효과를 일으킨다.

SSN으로 학생 레코드를 검색하여 브라우저에 표시하는 명령형 프로그램을 예를 들어 설명한다.

function showStudent(ssn){
  let student = db.find(ssn);
  if(student !== null){
    document.querySelector(`#${elementId}`).innerHTML =
      `${student.ssn},
      ${student.firstname},
      ${student.lastname}`;
  }
  else {
    throw new Error('Cannot found Student');
  }
}
showStudent('444-44-4444');

이 함수는 확실히 자신의 스코프를 벗어나 몇 가지 부수효과를 발생시킨다.
- 변수 db를 통해 데이터에 접근하는데, 함수 서명 signature에는 이런 매개변수가 없으니 이는 외부 변수입니다. 문제는 이 변수가 실행 중 언제라도 null을 참조하거나 호출 단계마다 상이한 값을 가리키면 결과가 완전히 달라지고 프로그램의 무결성이 깨질 수 있다.
- elementId는 전역 변수이다.
- HTML 요소를 직접 고친다. HTML 문서(DOM)는 그 자체로 가변적인, 전역 공유 자원이다.
- 예외 상황을 던지면 전체 프로그램의 스택이 풀리면서 종료된다.

위 함수는 외부 자원에 의존하여 코드가 유연하지 않고 다루기가 힘들며 테스트 역시 어렵다. 반면, 순수함수는 서명에 정규 매개변수 formal parameter (입력 집합)를 빠짐없이 명시하므로 코드를 이해하고 사용하는데 쉽다. 위 함수를 아래의 두 가지 포인트에 맞춰 개산해본다. 

- 긴 함수를 하나의 목적을 가진 짧은 함수로 각각 분리한다.
- 함수가 해야 할 작업에 필요한 인수를 모두 명시하여 부수효과 개수를 줄인다.

외부 저장소 및 DOM과 연동하면서 비롯되는 부수효과는 불가피하지만, 조금이라도 더 다루기 쉽고 주요 로직에 충실한 코드로 떼어낼 수 있다. 커링 currying이라는 FP 기법을 사용한다. 커링은 함수의 여러 인수를 부분적으로 나누어 세팅하는 것이다.

var find = curry((db, id) => {
  let obj = db.find(id);
  if(obj === null) {
    throw new Error('Cannot found Object');
  }
  return obj;
});

var csv = student => 
  `${student.ssn}, ${student.firstname}, ${student.lastname}`;

var append = curry((selector, inot) => {
  document.querySelector(selector).innerHTML = info;
})

- 재사용 가능한 컴포넌트 3개로 나뉘어 코드가 유연해졌다.
- 나누어진 함수를 재사용하면 신경 써서 관리할 코드가 줄기 때문에 생산성을 높일 수 있다.
- 프로그램이 할 일들을 high-level에서 단계별로 명확하게 보여주는 선언적 스타일을 따르므로 코드 가독성이 향상된다.
- HTML 객체와의 상호작용을 자체 함수로 빼내어 순수하지 않은 (불순한) 로직을 순수함수에서 배제했다. 커링 및 순수/불순한 부분을 다루는 방법은 4장에서 다룬다.

"참조 투명성과 치환성"

순수성 purity이란 함수의 인수와 결과 사이의 순수한 매핑 관계를 의미한다. 따라서 어떤 함수가 동일한 입력을 받았을 때, 동일한 결과를 내면 이를 '참조 투명한' 함수라고 한다.

var counter = 0;
function increment() {
  return ++counter;
}

increment는 상태적 함수로서 외부 변수 counter에 완전히 종속된 반환값을 내므로 참조 투명하지 않다. 참조 투명한 함수로 만들려면 이 함수가 의존하는 상태, 즉 외부 변수를 제거하고 함수 서명에 정규 매개변수로 명시해야 한다.

var increment = counter => counter + 1;

수정한 increment는 같은 입력에 같은 결과를 반환하는 안전한 함수이다. 이런 함수는 코드를 테스트하기 쉽고 전체 로직을 파악하는 것도 쉽다. 참조 투명성 혹은 등식 정합성 equational correctness은 수학에서 나온 용어지만 프로그래밍 언어의 함수는 수학 함수처럼 움직이지 않기 때문에 참조 투명성은 전적으로 개발자의 숙제로 남는다.

주어진 입력을 처리해서 결과를 내는 일련의 함수들로 임의의 프로그램을 정의한다고 하자. 의사 형식 pseudo form으로 나타내면 다음과 같다.

Program = [Input] + [func1, func2, func3, ...] -> Output

[func1, func2, func3, ...]이 모두 순수함수라면 결과를 바꾸지 않고 [val1, val2, val3, ...] 식으로 나열하여 프로그램을 쉽게 고칠 수 있다.

var input = [80, 90, 100];
var sum = (total, current) => total + current;
var total = arr => arr.reduce(sum);
var size = arr => arr.length;
var divide = (a,b) => a/b;
var average = (arr) => divide(total(arr), size(arr));
average(input); //-> 90;

위와 같이 순수 함수형 프로그램의 밑바탕에는 이러한 사고방식이 내재되어 있으며, 부수효과가 있는 함수라면 이런 일이 가능하지 않다.

"불변 데이터 유지"

자바스크립트의 모든 기본형 primitive type (원시 자료형)은 처음부터 불변이다.  그러나 배열 등의 객체는 불변이 아니어서 함수 인수로 전달해도 원래 내용이 변경되어 부수효과가 발생할 수 있다. 배열을 정렬하는 간단한 코드를 살펴본다.

var sortDesc = arr => {
  arr.sort(
    (a, b) => b - a;
  );
};

var arr = [1,2,3,4,5,6,7,8,9];
sortDesc(arr) //-> [9,8,7,6,5,4,3,2,1]

위 코드는 인수로 받은 배열의 원소를 내림차순으로 정렬한 뒤 반환하는 기능을 수행한다. 하지만 상태적 함수인 Array.sort는 원본 레퍼런스가 가리키는 배열의 원소를 정렬하는 부수효과를 일으킨다. 이것을 극복하는 방법은 다음 장 이후에 다룬다.

정리하면
함수형 프로그래밍은, 외부에서 관찰 가능한 부수 효과가 제거된 불변 프로그램을 작성하기 위해 순수함수를 선언적으로 평가하는 것이다.

"함수형 프로그래밍의 이점"

다음 세 가지 측면에서 나누어 살펴본다.

1. 복잡한 작업을 간단한 함수들로 분해한다.
  함수형 프로그래밍은 high-level에서 보면 사실상 분해와 합성 간의 상호작용이라 할 수 있다. 대개 함수형 사고는 어떤 작업을 논리적 하위작업 subtask으로 분할하는 행위부터 시작한다. 필요하다면 나누어진 하위작업을 더 단순한 순수함수로 분해해서 독립적인 작업 단위로 나타낼 수 있다.
  FP에서 모듈화 modularization은 단일성 singularity의 원리와 밀접한 관련이 있다. 함수는 각각 한 가지 목표만 봐라바야 한다는 것이다. 단순 함수를 엮어 붙이려면 반드시 입력 및 출력 형식을 서로 맞춰야 하는데, 순수성과 참조 투명성이 이런 사고방식을 갖도록 도와준다.
  합성 composition이라는 기법으로 함수를 구현하면, 두 함수를 합성하면 첫 번째 함수의 결과를 다음 함수에 밀어 넣는 새로운 함수가 탄생한다. 두 함수를 합성하려면 당연히 인수 개수와 형식이 서로 맞아야 한다. 합성 함수는 다른 함수를 인수로 받으므로 고계함수 high-order function이라고 한다.

2. 흐름 체인 fluent chain으로 데이터를 매끄럽게 처리한다.
  체인 chain은 같은 객체를 반환하는 순차적인 함수 호출이다. 체인도 합성처럼 코드를 간결 명료하게 작성하게 하고, 함수형은 물론 리액티브 자바스크립트 라이브러리에서도 활발히 쓰인다.
  복수 과목을 수강한 학생들의 평균 점수를 계산하는 프로그램을 예로 보자.
  let enrollment = [
    {enrolled: 2, grade: 100},
    {enrolled: 2, grade: 80},
    {enrolled: 1, grade: 89}
  ];
  명령형 프로그램으로 짜면 다음과 같을 것이다.
  var totalGrades = 0;
  var totalStudentsFound = 0;
  for(let i = 0; i < enrollment.length; i++){
    let student = enrollment[i];
    if(student !== null){
      if(student.enrolled > 1){
        totalGrades += student.grade;
        totalStudentsFound++;
      }
    }
  }
  var average = totalGrades / totalStudentsFound; // -> 90;

  Lodash JS라는 인기 있는 함수형 자바스크립트 도구에 구현된 고계함수를 통해 함수형 프로그램으로 리팩토링하면 다음과 같다.

  _.chain(enrollment)
    .filter(student => student.enrolled > 1)
    .pluck('grade')
    .average()
    .value(); //-> 90;

  함수 체인은 필요한 시점까지 실행을 미루는 lazy evaluation을 수행한다. 다른 데에선 전혀 쓸 일이 없는 일련의 코드를 전부 실행하지 않아 CPU 부하가 줄어들고 성능이 좋아진다. 이렇게 하면 다른 함수형 언어에 기본 탑재된 필요 시 호출 call-by-need동작을 효과적으로 모방할 수 있다.

  하지만 위 코드는 에러 처리 코드를 모두 무시하고 건너뛰었다. 예외를 던지는 건 부수효과를 유발한다. 순수 학문적 함수형 프로그래밍에는 예외가 존재하지 않지만, 실세계에서 예외를 완전히 배제하기는 어렵다. 몇몇 순수 함수형 디자인 패턴을 적용하면 표현성을 희생하지 않고도 탄탄한 에러처리 로직을 구사할 수 있다. 추후 5장에서 다룬다.

3. 리액티브 패러다임을 실현하여 이벤트 중심 코드의 복잡성을 줄인다.
  리액티브 패러다임의 가장 큰 장점은, 더 높은 수준으로 코드를 추상하여 비동기, 이벤트 기반 프로그램을 설정하느라 반복되는 boilerplate 코드는 잊고 비즈니스 로직에만 전념할 수 있게 해준다. 또 함수를 체인으로 묶고 합성하는 FP의 능력을 최대한 이끌어 낼 수 있다.
  리액티브 패러다임은 옵저버블 observable이라는 아주 중요한 장치를 매개로 움직인다. 옵저버블을 이용하면 데이터 스트림을 구독해서 원하는 연산을 우아하게 합성 및 체이닝하여 처리할 수 있다. 

"1장을 마치며"

- 순수함수를 사용한 코드는 전역 상태를 바꾸거나 깨뜨릴 일이 전혀 없기 때문에 테스트, 유지보수가 더 쉬운 코드를 개발하는 데 도움이 된다.
- 함수형 프로그래밍은 코드를 선언적으로 작성하므로 이해하기 쉽고 전체 애플리케이션의 가독성 역시 향상된다. 또 함수와 람다 표현식을 조합하여 깔끔하게 코딩할 수 있다.
- 여러 원소로 구성된 컬렉션 데이터는 map, reduce 같은 연산을 함수 체인으로 연결하여 매끄럽게 처리할 수 있다.
- 함수형 프로그래밍은 함수를 기본적인 구성 요소로 취급한다. 이는 일급/고계함수 개념에 기반을 두며 코드의 모듈성과 재사용성을 높인다.
- 리액티브/함수형 프로그래밍을 융합하면 이벤트 기반 프로그램 특유의 복잡성을 줄일 수 있다.
